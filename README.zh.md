## 什么是 Grok Imagine Video？

**Grok Imagine Video** 是 xAI 用文本、图像和源素材生成与编辑短视频的系统，声音也会作为结果的一部分生成。当前 `grok-imagine-video-1.5` 模型已通过 xAI API 正式开放。xAI 将此版本描述为在运动、物理连贯性、语音、音效、环境声和生成速度方面有所提升。

实用提示时，把每次请求当作镜头简报，而不是情绪板。写明可见动作、镜头行为、节拍时间、声音方案，以及必须保持稳定的内容。下面的案例把创作者提示词与结果并列，帮助你区分有效指导与装饰性措辞。

- **从文本或画面开始** - 根据文字场景生成，或让已经承载构图的图像动起来
- **描述可观察的运动** - 写清调度、动量、物体交互，以及每个动作造成的物理后果
- **写节拍，不写梗概** - 时序重要时，使用时间码或简短动作序列
- **让声音与场景一起生成** - 故事需要音频时加入对白、环境声、音乐或音效
- **编辑现有素材** - 重塑片段风格、改变场景条件，或添加、移除、替换可见元素
- **明确保护连续性** - 指出脸部、产品结构、布局、服装或背景中不能漂移的部分

**当前参考：** [xAI Video 1.5 release](https://x.ai/news/grok-imagine-video-1-5) · [xAI model documentation](https://docs.x.ai/developers/models/grok-imagine-video) · [Grok Imagine on ImagineVid](https://imaginevid.io/zh/grok-imagine)

### 把提示词变成镜头模板

可复用的视频提示词将场景变量与导演逻辑分开。替换主体、环境、台词或产品，同时保留经过验证的镜头路径、节拍结构、声音方案和连续性规则。

**模板模式：**
```
[DURATION + ASPECT RATIO]。[SUBJECT] 在 [SETTING] 中执行 [VISIBLE ACTION]。Camera: [FRAMING + MOVE]。Beats: [TIMED ACTIONS]。Audio: [DIALOGUE + FOLEY + AMBIENCE]。Preserve: [IDENTITY / PRODUCT / LAYOUT]。Avoid: [FAILURE MODES]。
```

从一个动作和一个镜头想法开始。只有在解决可见制作需求时才加入时间、音频和保留约束；然后每次生成只改变一个变量。
